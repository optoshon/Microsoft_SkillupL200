1. Environment Setup

Option A: Create New Conda Environment

conda create -n multimodal_rag python=3.10 conda activate multimodal_rag

Option B: Create Python venv Environment

# Windows: python -m venv multimodal_rag multimodal_rag\Scripts\activate # macOS/Linux: python -m venv multimodal_rag source multimodal_rag/bin/activate

Option C: Use Existing Environment

If you already have an environment created (e.g., myenv310), activate it:

# Conda: conda activate myenv310 # OR venv: myenv310/Scripts/activate # Windows source myenv310/bin/activate # macOS/Linux

Install Dependencies

After activating your environment, run:

pip install -r requirements.txt

Requirements include: vector DB (Chroma/FAISS), embeddings (Azure OpenAI), doc parsing (Docling), OCR (EasyOCR), reranking (sentence-transformers), and multimodal processing (LangChain).

⚠️ If you get package errors during script execution, uncomment the pip install line at the top of each .py file.

1.5. Environment Variables Setup

Create .env File

Create a .env file in your project root directory with your Azure OpenAI credentials:

AZURE_OPENAI_ENDPOINT=https://.openai.azure.com/ AZURE_OPENAI_API_KEY= AZURE_OPENAI_API_VERSION=2024-12-01-preview AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT=text-embedding-3-large AZURE_OPENAI_VISION_DEPLOYMENT=gpt-4o AZURE_OPENAI_LLM_DEPLOYMENT=gpt-4o AZURE_OPENAI_CHAT_DEPLOYMENT=gpt-4o

How to Get Azure OpenAI Credentials

  1. Go to Azure Portal
  2. Find your OpenAI resource (e.g., "tekframeworksllms")
  3. Click Keys and Endpoint in the left sidebar
  4. Copy Endpoint and API Key
  5. Paste into your .env file

Validate .env is Loaded

Run this command to verify all required environment variables are set:

python -c "import os; from dotenv import load_dotenv; load_dotenv(); for var in ['AZURE_OPENAI_ENDPOINT', 'AZURE_OPENAI_API_KEY', 'AZURE_OPENAI_API_VERSION', 'AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT']: print(f'{var}: {"✅" if os.getenv(var) else "❌"}') "

⚠️ Common Issues:

  • 401 Error: Invalid or expired API key. Regenerate in Azure Portal.
  • 404 Error: Deployment name doesn't exist. Check deployment names in Model deployments section.
  • .env not found: Place it in the same folder where you run the scripts from.

4. Execution Order (Mandatory)

Order Script Purpose
1 Read_File_Docling.py Extract text, tables, images from documents
2 Table_Embeddings_Langchain_vid.py Create structured embeddings from tables
3 Text_Embeddings_Langchain.py Chunk + embed document text
4 Image_VLM_Embeddings_Langchain_vid.py Convert images → structured JSON → embeddings
5 Multimodal_Ensemble_Retriever_Enterprise.py Unified search + rerank + LLM answer

5. Module Deep Dive

Module 1 — Document Extraction

Script: Read_File_Docling.py

What it does: Reads PDF, DOCX, PPT, and HTML documents. Extracts text, tables, and figures into separate, organized formats ready for processing.

Input required:

  • Documents in INPUT_DIR (PDF, DOCX, PPTX, HTML)
  • .env file with Azure OpenAI credentials (optional for extraction, required later)

Output produced:

  • docling_output//text/.docx — Extracted text as DOCX
  • docling_output//tables/table_001.csv — Tables as CSV files
  • docling_output//images/figure_001.png — Extracted figures as PNG
  • docling_output//tables.sqlite — All tables in SQLite DB
  • docling_output/manifest.json — Metadata index

Typical runtime: 30-120 seconds per document (depends on size and complexity)

Configuration required: Set INPUT_DIR and OUTPUT_DIR paths in script (search for "PUT YOUR PATH HERE")

Enterprise Advantage: Structured knowledge extraction across formats — no data loss.

Module 2 — Table Intelligence Engine

Script: Table_Embeddings_Langchain_vid.py

What it does: Reads CSV tables extracted by Docling. Applies intelligent row-based chunking with overlap. Converts to human-readable format (markdown or CSV) and creates embeddings for semantic search.

Input required:

  • docling_output/*/tables/*.csv — Tables from Module 1
  • .env with AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT

Output produced:

  • vector_db_table_chroma/ — Chroma vector database with table chunk embeddings
  • table_ingest_manifest.json — Metadata about ingested tables

Typical runtime: 15-60 seconds (depends on table size and count)

Configuration required: Set DOCLING_OUTPUT_DIR and PERSIST_DIR paths (search for "PUT YOUR PATH HERE")

Why special logic? Tables contain relational meaning (rows, columns, headers) not captured by plain text embeddings. Row-based chunking preserves table structure.

Common errors:

  • 401/404 Azure errors: Check .env file and deployment names
  • FileNotFoundError: Verify Module 1 completed successfully; check paths

Module 3 — Text Intelligence Engine

Script: Text_Embeddings_Langchain.py

What it does: Loads DOCX text extracted by Docling. Applies recursive character-based chunking to preserve semantic boundaries. Generates stable IDs for tracking. Embeds all chunks and stores in vector database.

Input required:

  • docling_output/*/text/*.docx — Documents from Module 1
  • .env with AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT

Output produced:

  • vector_db_text_chroma/ — Chroma vector database with text chunk embeddings

Typical runtime: 30-120 seconds (depends on document size)

Configuration required: Set DOCLING_OUTPUT_DIR and PERSIST_DIR paths. Adjust CHUNK_SIZE and CHUNK_OVERLAP if needed (search for "PUT YOUR PATH HERE")

Enterprise value: Precise semantic search across thousands of documents with stable chunk IDs for auditing and updates.

Module 4 — Image Intelligence Engine

Script: Image_VLM_Embeddings_Langchain_vid.py

What it does: Reads image figures extracted by Docling. Sends each to Azure OpenAI Vision (GPT-4V). Forces structured JSON output describing the image content. Embeds the JSON as text for semantic search without hallucinations.

Input required:

  • docling_output/*/images/*.png — Figures from Module 1
  • .env with AZURE_OPENAI_VISION_DEPLOYMENT and AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT

Output produced:

  • vector_db_image_vlm_chroma/ — Chroma vector database with image interpretation embeddings
  • image_vlm_manifest.json — Structured interpretations of each image

Typical runtime: 60-300 seconds (slower due to Vision LLM API calls; ~2-5 seconds per image)

Configuration required: Set DOCLING_OUTPUT_DIR and PERSIST_DIR paths (search for "PUT YOUR PATH HERE")

Example extracted intelligence:

  • Flowchart meaning and connections
  • Org chart hierarchy and roles
  • Chart values and trends
  • Diagram relationships and labels

Common errors:

  • Vision deployment not found: Ensure gpt-4o or gpt-4-vision is deployed in Azure
  • API rate limits: Large batches may hit rate limits; run in smaller batches

Module 5 — Enterprise Multimodal Retriever

Script: Multimodal_Ensemble_Retriever_Enterprise.py

What it does: Queries all 3 vector databases (text, tables, images) with the user query. Returns top results per modality. Deduplicates and optionally reranks using cross-encoder. Sends ranked context to LLM for final answer. Logs everything for audit and debugging.

Input required:

  • vector_db_text_chroma/ — From Module 3
  • vector_db_table_chroma/ — From Module 2
  • vector_db_image_vlm_chroma/ — From Module 4
  • .env with all Azure OpenAI deployments

Output produced:

  • outputs/run_YYYYMMDD_HHMMSS/ — Query artifacts including:
    • answer.txt — Final LLM-generated answer
    • context_blocks.csv — All retrieved chunks with scores
    • query_timeline.json — Per-stage timings and costs

Typical runtime: 10-30 seconds per query (depends on LLM response time)

Configuration required: Ensure all 3 vector DB directories exist (from Modules 2-4). Paths should match what those modules created.

Production Strength:

  • Observability logs and query timelines
  • Cost estimation hooks (configure in script)
  • Audit artifacts (all retrieved sources saved to CSV)
  • Deterministic deduplication and ranking
  • Interactive chat loop for live testing

Common errors:

  • Collection not found: Verify Modules 2-4 completed; vector DBs should exist
  • No results returned: Adjust TOP_K_PER_MODALITY or FINAL_CONTEXT_K parameters

8. Azure Deployment & Configuration

Create or Use Existing Azure OpenAI Resource

Step 1: Check if resource exists

  1. Go to Azure Portal
  2. Search for "OpenAI" or "Cognitive Services"
  3. If you see a resource (e.g., "tekframeworksllms"), proceed to Step 3
  4. If not, proceed to Step 2

Step 2: Create new Azure OpenAI resource (if needed)

  1. Click "Create a resource" → Search for "OpenAI"
  2. Click "Create" → Fill in details:
    • Resource Group: Create new or select existing
    • Region: East US, UK South, France Central, etc. (choose closest to you)
    • Name: e.g., "multimodal-rag-resource"
    • Pricing: Select Standard S0
  3. Click "Review + Create" → "Create" (wait 5-10 minutes)

Step 3: Deploy Models

Once your resource is ready:

  1. Click on your OpenAI resource
  2. In left sidebar, click Model deployments
  3. Click Create new deployment
  4. Deploy these models:
    Model Name Deployment Name Purpose Capacity
    text-embedding-3-large text-embedding-3-large Text embeddings (Modules 2, 3, 4) 1 (TPM: 240K minimum)
    gpt-4o gpt-4o Vision + Chat (Modules 4, 5) 1 (TPM: 30K minimum)
    gpt-4 gpt-4 Alternative chat model (optional) 1

Step 4: Copy Credentials

  1. Click on your OpenAI resource
  2. In left sidebar, click Keys and Endpoint
  3. Copy Endpoint (URL starting with https://...)
  4. Copy Key 1 or Key 2
  5. Paste into your .env file:
AZURE_OPENAI_ENDPOINT=https://your-resource.openai.azure.com/ AZURE_OPENAI_API_KEY=your-key-here AZURE_OPENAI_API_VERSION=2024-12-01-preview AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT=text-embedding-3-large AZURE_OPENAI_VISION_DEPLOYMENT=gpt-4o AZURE_OPENAI_LLM_DEPLOYMENT=gpt-4o AZURE_OPENAI_CHAT_DEPLOYMENT=gpt-4o

Step 5: Validate Connection

Run this test to verify everything is working:

python -c " from openai import AzureOpenAI import os from dotenv import load_dotenv load_dotenv() client = AzureOpenAI( api_key=os.getenv('AZURE_OPENAI_API_KEY'), api_version=os.getenv('AZURE_OPENAI_API_VERSION'), azure_endpoint=os.getenv('AZURE_OPENAI_ENDPOINT') ) response = client.chat.completions.create( model=os.getenv('AZURE_OPENAI_CHAT_DEPLOYMENT'), messages=[{\"role\": \"user\", \"content\": \"Say OK\"}] ) print('✅ Azure connection successful:', response.choices[0].message.content) "

Expected output: ✅ Azure connection successful: OK

Troubleshooting:

  • 401 Unauthorized: Check API key is correct and not expired. Regenerate in Azure Portal if needed.
  • 404 Not Found: Check deployment names. Go to Model deployments and verify names match your .env.
  • Connection timeout: Check your internet connection and Azure resource region.

10. Troubleshooting & FAQ

Q: How do I verify .env is loaded correctly?

A: Run this command in your terminal:

python -c "import os; from dotenv import load_dotenv; load_dotenv(); for k in ['AZURE_OPENAI_ENDPOINT', 'AZURE_OPENAI_API_KEY', 'AZURE_OPENAI_API_VERSION', 'AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT']: print(f'{k}: {os.getenv(k)[:30] + \"...\" if os.getenv(k) else \"NOT SET\"}') "

Q: I get "401 Access denied" error

A: Your Azure API key is invalid or expired. Fix it:

  1. Go to Azure Portal → Your OpenAI resource
  2. Click Keys and Endpoint
  3. Click Regenerate Key 1 (or Key 2)
  4. Copy the new key and update your .env file
  5. Run your script again

Q: I get "404 Resource not found" error

A: The deployment name doesn't exist. Fix it:

  1. Go to Azure Portal → Your OpenAI resource
  2. Click Model deployments in left sidebar
  3. Check the exact names of your deployments
  4. Update your .env file:
    • AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT= (copy exact deployment name for embeddings)
    • AZURE_OPENAI_VISION_DEPLOYMENT= (copy exact deployment name for gpt-4o)

Q: How do I check if my Azure resource is still active?

A: Go to Azure Portal → Your resource → Overview. Look for status indicator. It should be green ("Running"). If red or deleted, you'll need to create a new resource.

Q: Module X says "FileNotFoundError: docling_output not found"

A: You haven't run Module 1 (Read_File_Docling.py) yet. Run it first to extract documents. Also check:

  1. Set correct INPUT_DIR path in Read_File_Docling.py (search for "PUT YOUR PATH HERE")
  2. Put your documents (.pdf, .docx, etc.) in that folder
  3. Run: python Read_File_Docling.py
  4. Wait for it to complete (check for ✅ Done message)
  5. Then run Module 2

Q: How do I clear vector databases and re-index?

A: Delete the Chroma folders and re-run the indexing scripts:

# Windows: rmdir /s vector_db_text_chroma rmdir /s vector_db_table_chroma rmdir /s vector_db_image_vlm_chroma # macOS/Linux: rm -rf vector_db_text_chroma rm -rf vector_db_table_chroma rm -rf vector_db_image_vlm_chroma # Then re-run modules 2-4 python Table_Embeddings_Langchain_vid.py python Text_Embeddings_Langchain.py python Image_VLM_Embeddings_Langchain_vid.py

Q: Can I run just one module to test?

A: Yes, but with dependencies:

  • Module 1: Independent. Can run alone.
  • Module 2, 3, 4: Each requires Module 1 output first.
  • Module 5: Requires Modules 2-4 output. Run all 3 before running Module 5.

Q: A script hangs or times out. What do I do?

A: Common causes:

  • Large documents: Docling can take a long time. Start with smaller documents (< 50 pages).
  • API rate limits: Azure OpenAI has rate limits. Wait 1-2 minutes and retry.
  • Network issues: Check internet connection. Temporarily move closer to router.
  • Infinite loop: If script doesn't progress after 5+ minutes, press Ctrl+C to stop. Then debug the script.

Q: How do I install missing packages if I get import errors?

A: Uncomment the !pip install line at the top of each script and run it again. Or manually:

pip install -r requirements.txt
If a specific package fails, install it individually. For example:
pip install langchain-openai pip install docling pip install sentence-transformers

Q: My environment already exists (e.g., myenv310). How do I use it?

A: Activate it instead of creating new one:

# Conda: conda activate myenv310 # OR venv: myenv310\Scripts\activate # Windows source myenv310/bin/activate # macOS/Linux # Then install dependencies: pip install -r requirements.txt
